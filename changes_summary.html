AI Reading Reports Admin Panel

Changes Summary & Flow Documentation

This document outlines the modifications implemented to add the read-only AI Reading Reports monitoring surface to the Divya Sadhana Admin Panel.

1. Flow Overview

  1. User Request Submission: End users upload a photo (Face or Palm image) and answer custom focus questions (e.g. Focus area: Career). This request is logged in the backend.
  2. AI Report Generation: The Django backend sends the input and template context to the Generative AI vision models, saving both a short teaser report and a full report.
  3. Payment: Users pay via Razorpay. Once payment is captured and verified, the report becomes "unlocked" for the user.
  4. Admin Inspection: Superusers login to the admin panel and navigate to the "AI Readings" menu to view the request list.
  5. Ungated Override: In the detailed view, admins can see both the teaser and the full HTML reports regardless of whether the user has completed their payment. This allows transaction monitoring and troubleshooting.

2. Admin Panel Interface

A. Readings List Page (/ai-readings)

  • Interactive Filters: Allows filtering requests by Service Kind (Face Reading vs. Palm Reading) and Lifecycle Status (Pending, Processing, Succeeded, Failed, Cancelled).
  • Search: Text search querying request numbers and nested user email addresses.
  • Cache Hits: Clearly distinguishes if a request was served fresh or pulled from an image cache (marked with cache hit status).

B. Details Page (/ai-readings/[id])

  • Client Information: Shows full name, email, phone number, and active state of the submitting user.
  • Context Card: Displays chosen answers and price details.
  • Attachment Image Preview: Direct thumbnail displaying the uploaded photo.
  • Isolated Preview Frames: Embedded preview sandboxes (iframes) to view HTML reports without CSS leakage affecting the dashboard layout.
  • Observability Dashboard: Displays model IDs, processing duration (in seconds), input/output token metrics, and API request IDs.
  • Payment Ledger: Lists all payment attempts with internal references, Razorpay order/payment IDs, capture times, and status markers.

3. Summary of File Edits

1. Schema Definition

Created src/schemas/ai-readings.schema.ts defining types and Zod parser constraints for AiReading, AiReadingReport, and list arrays.

2. API Service Layer

Created src/services/ai-readings.service.ts to handle the HTTP network calls to GET /api/admin/ai-readings/ and GET /api/admin/ai-readings/{id}/.

3. React Query Hooks

Created src/hooks/queries/useAiReadingsQuery.ts wrapping the services inside useAiReadingsListQuery and useAiReadingQuery hooks.

4. Columns Config

Created src/hooks/tables/useAiReadingsTableColumns.tsx configuring list headers, user metadata sub-labels, prices, and status indicators.

5. Routing Pages
  • src/app/(admin)/ai-readings/page.tsx: Renders filters, search, and paginated datatable.
  • src/app/(admin)/ai-readings/[id]/page.tsx: Renders core metrics, previews, tokens, and payment ledger tables.
6. Navigation Layout

Modified src/components/layout/Sidebar.tsx to add the navigation item linking to the new page.
